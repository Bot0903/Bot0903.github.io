King's Meal

This game was made as a part of the Capstone project for the Game Design and Development program. I worked on this project with two other students, I was in charge of coding and gameplay design. I looked to test myself with the coding element of this project and tested new elements that I have not worked with before.

Notable elements of this project that I worked on:

- Turn based combat system
- Two modes: exploration and combat
- Player and enemy navigation
- Object oriented item inventory system

Turn-based Combat

Very early in this project we knew we wanted to make something turn-based. This turn system uses an array to add and remove enemy objects from the turn order when the player encounters and defeats enemies. Both player and enemies have two moves per turn, which in hindsight should have been displayed.

Exploration and Combat

For this game we wanted the ability to roam freely without the limitation of combat movement while not in combat with an enemy. Because of this the turn based system was made with that in mind allowing for there to be two states that the player can be in. This is done by checking the turn-based array for an enemy object if the player is the only one in the array that means there is no combat.

Player and Enemy Navigation

Using the nav mesh feature in Unity I was able to create navigation for both the player and the enemies within the game. Using the nav mesh I was able to make preferred paths for the player within the swamp biome as well.

Object Oriented Items

Object oriented design is something that has become very important within games and so this was my first attempt at using this type of system.
