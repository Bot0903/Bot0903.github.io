Sky Palace Hotel

This is a text based game created on Twine during my time in university. This project was done on my own. This game explored some of the abilities of the Twine system. Twine is a platform commonly used for text based games and by default uses a language called Harlowe, this project was made using Harlowe.

Notable elements of this project

- Branching Story Design
- Items
- Dynamic Room Interaction

Branching Story Design

The game features many branching options throughout the duration of the game, these options can sometimes depend on how the player has progressed or if the player has acquired a certain item or not.

Items

There are items to be found throughout the game that serve different functions. This item system is however an illusion; there is no actual inventory, since there are not that many items in the game I was able to have the items behave like triggers. If I were to remake the game now I would look to make the game using JavaScript instead and perhaps create a proper inventory system.

Dynamic Room Interaction

Using the Harlowe language provided by Twine there is a way to track the number of times a player has visited a certain room. With this mechanic I was able to create some interesting rooms within the game. Some rooms will hold secrets that will only reveal itself if the player has visited a room enough times, or if a player is stubborn about a prompt the player may receive dynamic interaction.
